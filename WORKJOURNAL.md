## 2026-06-26 — fluid, role-aware card animation (on `main`)
- Reworked the plex recenter animation so card appearance/disappearance reads as *natural* and *ontology-aware*, and the whole motion feels calmer. Brainstormed first (two decisions locked with the user: **motion model = grow-from / retract-to the active thought**, symmetric; **feel = gentle, all-together**, no cascade).
- **Card reuse already existed** (elements keyed by lowercased name → CSS `transform` transition glides a clicked card to center). The eagerness came from *new* and *dropped* cards, which this pass fixes:
    - **New cards now grow out of the active thought's gate on their own side** (parent→top, child→bottom, jump→left, sibling→right) while fading in, instead of all spawning at the dead center `{0,0}` at full opacity. Origin comes from a new pure helper **`focusGatePoint(zone)`** in `edges.js` that reuses the existing `GATES[zone].focus` mapping + `gatePoint`, so the emergence point can never drift from where the connector actually attaches.
    - **Dropped cards now retract back into the active thought along their OLD direction** (`el._zone`) while fading out — the mirror of the entry — instead of fading in place.
    - **Reused cards** keep gliding directly old→new (now via an explicit early-`continue` branch in `setGraph`).
- **Softer timing:** `320→420ms`, easing `cubic-bezier(0.4,0,0.2,1)` → `(0.22,1,0.36,1)` (fast out, slow settle so the graph *unfolds*), fade `0.3s`. Added a `prefers-reduced-motion: reduce` block that drops the glide/fade entirely.
- **Adversarial review** (multi-agent, 4 lenses: state-machine, geometry/matrix-invariant, regressions/interruption, CSS; each finding refuted by two skeptics). The core invariant — that `getComputedStyle().transform`'s `m41/m42` still recover the card **center** so edges/handles/drag stay aligned — was proven to hold. The **only** confirmed issue: an early experiment added a `scale(0.9)` grow, but `gatePoint` uses *unscaled* half-dims, so the connector overshot a still-shrunken card's border by up to ~10.4px (jump/sibling) mid-tween. **Resolved by dropping the scale** — the grow-from-active-thought feel is fully carried by the translate-origin + fade, so removing scale eliminates the gap (connectors dock exactly every frame), keeps the approved motion, and leaves no unused `scale` param behind.
- New: `focusGatePoint` unit tests in `edges.test.js`. Tests **68→70**, production build clean.
- **Follow-up (live):** 420ms still read as too eager, so doubled the recenter motion to **840ms** glide + **0.6s** fade (`TRANSITION_MS` and the CSS `transform`/`opacity` durations kept in sync so the leaving-card DOM removal still matches the transition end). Hover transitions (`background`/`border-color` 0.15s) left untouched — they're not part of the recenter motion.
- **Follow-up (connected-card semantics):** replaced the role-gate emergence origins with anchors tied to the *connected* card — appearance/disappearance now references the card a link belonged to, not the new center:
    - **Disappearing cards collapse INTO the old active thought's NEW position.** On recenter the old active thought demotes to a parent/jump/sibling and slides there; a dropped card animates to that spot (looked up in the fresh `layout.nodes`) and fades as it arrives — they meet coincident because they share the easing/duration. Worked example: activating a child makes the old active thought's **jumps and siblings move up and toward the center** and fade into the card that just became a parent (sole-parent ⇒ exit target `(0,-BAND_Y)`). Falls back to center if the old active thought is absent from the new graph.
    - **Appearing cards emerge FROM the activating card** (the new active thought) at its current on-screen position (`liveCenterOf` captured before the loop recenters it), falling back to center on a fresh load. New cards grow out of the card you clicked.
    - Removed the now-unused `focusGatePoint` helper + its tests. Verified by a 3-lens adversarial review (exit-anchor incl. the worked example, enter-anchor + interruption/races, regression/dead-code): **0 findings**. Tests 70→68 (the two `focusGatePoint` tests removed), build clean.

## 2026-06-26 — terminology pass (docs only)
- Pinned down the project's vocabulary so docs, comments, and UI copy stop drifting between synonyms (*note/page/focus/node/relationship*). Added a canonical **`## Terminology`** glossary to `CLAUDE.md` and propagated it through the canonical docs + this journal.
- **Canonical terms:** **thought** (a piece of user content — a Logseq page/journal backed by a file), **card** (its on-screen box), **link** (a connection between two thoughts; each kind is many-to-many), **connector** (the line drawn for a link), **active thought** (the centered one; only its links show), **activate** (make a thought active — click its card, open its page, or create it), and **recenter** (the camera glide that plays on activate). Link kinds by position: **parent** above, **child** below in two columns, **jump** left, **sibling** right (computed).
- **Scope:** revised `CLAUDE.md`, `README.md`, `BACKLOG.md`, and `WORKJOURNAL.md`; left `docs/superpowers/*` (dated specs/plans) untouched as historical records.
- **Two deliberate splits baked into the glossary:** *connector* is the prose/visual term, *edge* stays the code term (`edges.js`, `computeEdges`, `edge-hit.js`) — link↔connector mirrors thought↔card; and *recenter* names the motion while *activate* names the action. "Page" is kept wherever it means the Logseq substrate (page properties, `getPage`, the first block).
- **Code symbols preserved verbatim** (`NODE`/`nodeAdjacency`/`focusGatePoint`/`GATES[zone].focus`/`layout.nodes`/`role:neighbor`/…); the glossary records each prose↔symbol mapping so docs still trace to source. Verified no prose stragglers remain; docs-only, no code or tests touched.
