## 2026-06-27 — connectors between all visible cards (secondary links, faded)
- The synapses now draws a connector for **every declared parent/child/jump link between two visible cards**, not just links touching the active thought. Links that don't involve the active thought ("secondary" links) render **faded** (`globalAlpha 0.4`, under the primary edges) and are **display-only** — not hit-tested, so the active thought's connectors keep today's hover-highlight + unlink × and the secondary ones add no clutter/interaction. Shared core → both Logseq and Obsidian.
- **Data was already in hand:** `nodeAdjacency` already returns each visible card's full `{parents,children,jumps}`, fetched per recenter and (until now) used only for handle states. Gate sides follow the **dominant axis** between the two card centres (horizontal vs vertical S-curve) so curves read naturally between arbitrary positions. `role` only drives colour (`jump` → jump colour, else normal); `remove: null` (display-only). `drawEdges` gained a `secondary` param drawn first at `SECONDARY_ALPHA` (tunable).

## 2026-06-27 — Obsidian breadcrumb fixes (history visibility + crumb re-activation)

In Logseq the synapses root is an iframe `<body>` that fills the sidebar, so `100vh` == the sidebar (correct); in Obsidian the root is the leaf `contentEl`, shorter than the window, so `100vh` overflowed the leaf and the bottom breadcrumb spilled past it, clipped by `.synapses-root { overflow:hidden }`. Fix: fill the **container** (`height: 100%`) instead of the viewport, and add `<style>html,body{height:100%}</style>` to `packages/logseq-plugin/synapses.html` so the iframe body resolves a real height for the `%` chain (Obsidian's flex leaf already provides one — Logseq output is byte-identical).

Re-routed to the normal activation path `goto(name)` → `histPush` → `pushEntry`, which already de-dupes and moves the entry to the most-recent (right-most) slot (covered by `history.test.ts`). Back/forward arrows (◀/▶) intentionally stay pointer-move — that's their meaning.

## 2026-06-26 — terminology pass (docs only)
- Pinned down the project's vocabulary so docs, comments, and UI copy stop drifting between synonyms (*note/page/focus/node/relationship*). Added a canonical **`## Terminology`** glossary to `CLAUDE.md` and propagated it through the canonical docs + this journal.
